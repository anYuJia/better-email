# Better Email

Better Email 是一个从零开始实现的跨平台轻量邮箱客户端原型，目标是用 Tauri + Rust + SQLite 达成低内存、小体积和接近完整邮箱 App 的体验。

## 当前状态

已实现第一阶段本地闭环，并扩展了第二批常见邮箱功能：

- 三栏邮箱界面：文件夹、邮件列表、线程列表、阅读面板，并支持拖拽调整侧边栏/邮件列表宽度、持久化布局和一键恢复默认宽度；视觉采用 Apple/Foxmail 的平面桌面工作区，并借鉴 QQ 的紧凑搜索、头像列表、中性选中态和图标操作节奏，不照搬聊天气泡；高频回复、回复全部、转发、归档、已读切换和删除直接可见，低频整理与安全动作按组收进菜单。账号设置使用分类侧栏和连续内容区，连接测试、保存设置固定在窗口顶部，窄屏自动收为带提示的图标按钮，诊断与备份仍留在对应分类。前端已拆分 `Sidebar`、`AccountSwitcher`、`SidebarFolderNavigation`、`MessageListPane`、`ReaderPane`、设置组件、`useContactManagement`、`useOAuthFlow`、领域类型和配置模块；联系人筛选/CRUD/VIP/合并与 OAuth2 PKCE/回调/token 交换/刷新分别由独立领域 Hook 管理，避免原生下拉和单文件继续膨胀。
- SQLite 本地存储和种子邮件，包含账号、文件夹、邮件、标签、附件元数据。
- 搜索、保存搜索快捷方式、图标式高级搜索条件菜单、刷新、未读/星标/附件过滤、线程/会话入口、`from:`/`to:`/`cc:`/`bcc:`/`subject:`/`label:`/`account:`/`mailbox:`/`folder:`/`filename:`/`attachment:`/`after:`/`before:`/`has:`/`is:` 搜索操作符、标星、已读/未读、稍后处理/取消稍后、到期稍后邮件自动恢复收件箱、归档、删除、自定义文件夹创建/重命名/删除、移动到系统或自定义文件夹、手动标为垃圾邮件/不是垃圾邮件、废纸篓恢复/永久删除/清空、本地 `.eml` 导入和邮件 `.eml` 导出，并支持列表批量选择后批量已读/未读、星标、归档、删除、移动和打标签；归档、删除、移动、已读/未读、星标、标签、垃圾邮件和稍后处理等常见误操作会弹出撤销 snackbar，使用轻量状态快照恢复原文件夹、已读、星标、稍后时间和标签。
- 桌面快捷键：`Cmd/Ctrl + K` 打开命令面板，`?` 或 `Cmd/Ctrl + /` 打开快捷键帮助面板，并支持搜索聚焦、写邮件、上下切换、回复/回复全部/转发、星标、已读、归档和删除。
- 标签展示、邮件标签增删和批量打标签；阅读面只显示当前邮件已有标签，完整标签选择折叠到“标签”菜单，批量标签入口折叠到批量“更多”菜单。
- 撰写窗口默认只保留收件人、主题和正文主路径；发件账号/身份、抄送/密送、富文本 HTML、身份签名、稍后发送、附件和本地写信模板收进“工具”折叠区，支持模板保存、插入、删除、写作过程自动保存与刷新后恢复、阅读面板内联快速回复、联系人自动补全、联系人中心搜索、一键给联系人写信、从命令面板按联系人发起写信、联系人新建/编辑/删除、别名管理、VIP 后端持久化、手动合并和重复联系人合并建议，回复、回复全部、转发、保存草稿、从草稿箱继续编辑；发送按钮支持关闭或 5/10/20/30 秒撤销发送延迟（默认 10 秒），倒计时内可一键撤回到草稿箱，到期后自动进入持久化 SMTP 后台任务。
- 多账号与账号设置：新增账号、统一邮箱/单账号范围切换、写信发件账号选择、发件身份/别名管理、Reply-To 和身份级签名、统一邮箱下按待同步优先级聚合扫描多账号、IMAP/SMTP 主机、认证方式（应用专用密码/授权码或 OAuth2 Token）、OAuth2 PKCE 授权页启动、授权会话恢复记录、回调授权码记录、授权码 token 交换入 Keychain、refresh token 自动刷新和 XOAUTH2 登录、同步策略、账号级远程图片策略、按发件人/域名持久化的远程图片信任列表、签名。
- Gmail、Outlook、QQ 邮箱、网易 163 常见 IMAP/SMTP 服务商预设和兼容性矩阵，并按服务商预填推荐认证方式。
- 设置页可导出脱敏诊断 JSON，用于真实账号兼容性排查，不包含密码、token、授权 URL 或完整邮箱地址；也支持本地 JSON 备份/恢复，覆盖账号配置、文件夹、邮件、标签、附件元数据、规则、发件箱和同步记录，系统 Keychain 凭据不会写入备份。
- 设置页可按服务商记录真实账号验证结果：IMAP、SMTP、OAuth2/XOAUTH2、诊断导出状态和备注会保存在本地。
- 系统 Keychain 凭据保存/检查/删除，敏感信息不写入 SQLite。
- `mail-parser` MIME 解析、`ammonia` HTML sanitizer、受控 HTML 安全预览、正式阅读面安全渲染、默认阻止远程图片并支持信任发件人/域名后重渲染、阻止发件人并自动移入垃圾邮件、钓鱼链接域名不一致/IP/登录验证路径提示、附件名/数量识别。
- 连接测试、IMAP 登录文件夹发现、OAuth2/XOAUTH2 登录、远端文件夹映射/UID 游标、邮件头增量同步、按同步策略触发的轻量后台同步、远端正文按需拉取。
- 新邮件系统通知：后台/手动同步导入新邮件后申请系统通知权限并发送摘要，支持本地免打扰时段、VIP 发件人、仅 VIP 提醒、账号静音、重点账号提醒和系统未读角标，侧边栏展示最近同步、通知和角标状态。
- SQLite 持久化后台任务队列：手动/定时邮件头同步、发件箱发送演练和真实 SMTP 发送进入统一串行队列，任务状态可跨刷新/重启恢复，侧边栏展示最近任务状态；同类 queued/running 任务统一去重，延迟发送、稍后发送和失败重试只使用最早到期项的单个前端定时器触发。
- 正文拉取后会缓存附件元数据，附件文件可按需下载；真实 IMAP 下载优先用 BODYSTRUCTURE 定位 MIME part，再通过 BODY.PEEK part/chunk 分段写入本地临时文件，默认 25 MB 下载上限，完成后记录本地路径、交给系统默认应用打开，并支持系统保存对话框另存为。
- 写信附件通过系统文件选择器或拖拽投放区记录文件元数据/可用本地路径，不把附件内容读入前端内存；附件会随草稿、发件箱和已发送消息保存元数据，富文本正文会清洗后保存为安全 HTML，并在发件箱真实 SMTP 发送时构建 `multipart/alternative` 或含附件的 `multipart/mixed` MIME。
- 发件箱队列、稍后发送、撤回到草稿箱、真实 SMTP 发送入口、OAuth2/XOAUTH2 发信认证、失败重试状态和 `next_attempt_at` 下次尝试时间。
- 联系人中心、联系人新建/编辑/删除、侧栏右键直达联系人编辑、别名/VIP 管理、手动合并、重复联系人合并建议、文件夹右键全部已读/清空废纸篓、自动规则执行、可视化规则构建器、规则新增/编辑/启停/删除、线程摘要和线程阅读视图等常见邮箱组织能力。
- Rust 数据层、协议、SMTP 和凭据边界单元测试，前端工具/服务商矩阵测试、Chrome headless UI 回归、桌面通知/角标权限配置、构建与 Clippy 验证。

完整设计见 [docs/DESIGN.md](docs/DESIGN.md)，验证记录见 [docs/VALIDATION.md](docs/VALIDATION.md)。

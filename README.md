# Better Email

Better Email 是一个从零开始实现的跨平台轻量邮箱客户端原型，目标是用 Tauri + Rust + SQLite 达成低内存、小体积和接近完整邮箱 App 的体验。

## 当前状态

已实现第一阶段本地闭环，并扩展了第二批常见邮箱功能：

- 三栏邮箱界面：文件夹、邮件列表、线程列表、阅读面板，并支持拖拽调整侧边栏/邮件列表宽度、持久化布局和一键恢复默认宽度；视觉采用 Apple/Foxmail 的平面桌面工作区，并借鉴 QQ 的紧凑搜索、头像列表、中性选中态和图标操作节奏，不照搬聊天气泡；高频回复、回复全部、转发、归档、已读切换和删除直接可见，低频整理与安全动作按组收进菜单。设置中心使用固定分类侧栏和 12 个独立分页，一次只渲染当前设置，并提供上一项/下一项导航；900px 以下以带分组的单一分类选择器替代侧栏，避免横向导航拥挤，也不会退化成难辨认的纯图标分页。普通设置入口稳定回到账号首页，联系人编辑仍直达联系人分页；窄屏侧栏底部仍保留设置、快捷键和命令三个紧凑图标入口。保存设置固定在窗口顶部，服务器测试只在账号、服务商、认证和同步等连接相关页面显示；窄屏标题栏操作自动收为带提示的图标按钮。账号、服务商、认证、发送、通知、隐私和身份页面已拆为独立组件，新增账号、OAuth2、同步、备份和安全预览进入页面即可直接操作，仅真正低频的技术详情继续折叠。认证区由服务商感知的独立安全凭据面板承载，针对网易 163、QQ、OAuth2 与自定义邮箱显示正确凭据类型，支持显示/隐藏、快速清空、仅保存和“保存并验证”，保存成功后立即清空输入框且只写入系统凭据库。四步诊断清晰区分服务器连接、系统凭据、IMAP 和 SMTP，针对 Gmail、Outlook、QQ、网易 163 和自定义邮箱给出恢复建议，原始端点结果默认收进“技术详情”，且不会显示或导出授权码与 Token；“一键验收”可顺序执行服务器检查、登录验证、文件夹发现和首轮邮件头同步，不发送邮件或修改远端状态。SMTP 单独失败时仍继续验证 IMAP 收信链路并显示“部分通过”，只有 IMAP 不可用时才停止后续步骤；同步页首屏的“发送与回写验收”只会生成发给当前账号、带唯一编号的验证草稿，不自动发送、不自动添加附件或定时发送，用户仍需在撰写器中检查并手动确认。最近验证编号按账号保存在本机，状态面板跨草稿/发件箱/已发送/收件箱聚合 SMTP、Sent 留档、自发自收、附件读取和远端可回写状态，并可刷新或直接定位已发送与收件副本；独立回写验收面板按已读、星标、归档、恢复四步引导，在前置邮件条件满足后才解锁操作。前端已拆分 `Sidebar`、`AccountSwitcher`、`SidebarFolderNavigation`、`MessageListPane`、`ReaderPane`、`AccountRemovalPanel`、`CredentialSecuritySettings`、`ProviderWriteValidationSettings`、`ProviderWritebackValidationPanel`、独立设置页面、`useContactManagement`、`useOAuthFlow`、`useProviderWriteValidation`、领域类型和配置模块；联系人筛选/CRUD/VIP/合并、账号安全移除、OAuth2 PKCE/回调/token 交换/刷新与写入验收跟踪分别由独立组件或领域 Hook 管理，避免原生下拉和单文件继续膨胀。
- 邮件与会话视图加载已拆入 `useMailboxData`：集中管理账号/文件夹/搜索/筛选请求、分页探测、刷新竞态、空视图清理、未读回退和首屏就绪标记；`App.tsx` 继续只保留页面编排与用户操作。
- 写信窗口、设置框架及各设置页、命令面板和快捷键帮助使用 `React.lazy` 按需加载，并用统一轻量浮层反馈首次打开状态；2026-07-11 构建样本的首屏直接加载资源为 472.1 KB，另有 213.6 KB 低频资源按需加载，完整前端产物为 686.1 KB，相比拆分前约 619.8 KB 的主 JS + CSS，首屏资源下降约 24%。`npm run bench` 会分别报告 `frontend_initial_load` 与 `frontend_deferred_assets`，避免用整个 `dist` 目录误判启动负担。
- SQLite 本地存储和种子邮件，包含账号、文件夹、邮件、标签、附件元数据。
- 搜索支持当前文件夹、当前账号全部文件夹、全部账号全部文件夹三档范围，邮件与会话视图共享同一范围，保存搜索会同时保存并恢复查询、筛选和范围；另有图标式高级搜索条件菜单、刷新、未读/星标/附件过滤、线程/会话入口、`from:`/`to:`/`cc:`/`bcc:`/`subject:`/`label:`/`account:`/`mailbox:`/`folder:`/`filename:`/`attachment:`/`after:`/`before:`/`has:`/`is:` 搜索操作符、标星、已读/未读、稍后处理/取消稍后、到期稍后邮件自动恢复收件箱、归档、删除、自定义文件夹创建/重命名/删除、移动到系统或自定义文件夹、手动标为垃圾邮件/不是垃圾邮件、废纸篓恢复/永久删除/清空、本地 `.eml` 导入和邮件 `.eml` 导出，并支持列表批量选择后批量已读/未读、星标、归档、删除、移动和打标签；会话列表支持右键已读、星标、归档、移动、标签和删除，阅读页默认选中最新邮件，并常驻会话级星标、回复/回复全部、转发、归档、已读和删除动作；归档、删除、移动、已读/未读、星标、标签、垃圾邮件和稍后处理等常见误操作会弹出撤销 snackbar，使用轻量状态快照恢复原文件夹、已读、星标、稍后时间和标签。
- 稍后处理使用统一时间选择器，提供今天晚些时候、明天上午、本周末、下周一和自定义日期时间；邮件右键菜单、阅读页更多菜单、命令面板、批量操作栏和线程右键菜单共用同一套交互，批量操作保留撤销快照。
- 联系人设置支持本地 vCard 3.0/4.0 导入与 vCard 4.0 导出；导入使用系统文件对话框，只接受 UTF-8 文本且限制为 5 MB，按主邮箱去重，合并别名与 VIP 状态，不覆盖已有有效名称。联系人页高频操作保持直接可见，620px 以下收紧为带 tooltip 和无障碍名称的图标按钮，不额外藏入多级菜单。
- 会话摘要与普通邮件列表使用同一视图作用域：账号、当前文件夹、搜索词、未读/星标/附件筛选、排序方式和分页上限会同时传给后端；支持最新优先、最早优先、发件人/参与者和主题排序，偏好保存到本机并对非法值安全回退到最新优先。切换到空账号或未选择文件夹时立即清空旧会话，避免跨账号、跨目录混入和短暂闪现。
- 桌面快捷键：`Cmd/Ctrl + K` 打开命令面板，`?` 或 `Cmd/Ctrl + /` 打开快捷键帮助面板，并支持搜索聚焦、写邮件、上下切换、回复/回复全部/转发、星标、已读、归档和删除。
- 标签展示、邮件标签增删和批量打标签；阅读面只显示当前邮件已有标签，完整标签选择折叠到“标签”菜单，批量标签入口折叠到批量“更多”菜单。
- 撰写窗口默认只保留收件人、主题和正文主路径；发件账号/身份、抄送/密送、富文本 HTML、身份签名、稍后发送、附件和本地写信模板收进“工具”折叠区，支持模板保存、插入、删除、写作过程自动保存与刷新后恢复、阅读面板内联快速回复、联系人自动补全、联系人中心搜索、一键给联系人写信、从命令面板按联系人发起写信、联系人新建/编辑/删除、别名管理、VIP 后端持久化、手动合并和重复联系人合并建议，回复、回复全部、转发、保存草稿、从草稿箱继续编辑；回复、回复全部和快速回复会继承原邮件 `Message-ID` / `References`，持久化并发送标准 `In-Reply-To` / `References` 头，避免跨客户端回复被拆成新会话；发送按钮支持关闭或 5/10/20/30 秒撤销发送延迟（默认 10 秒），倒计时内可一键撤回到草稿箱，到期后自动进入持久化 SMTP 后台任务。
- 多账号与账号设置：新增账号会校验邮箱并创建默认系统文件夹和默认发件身份，可立即切换到新账号；支持设置唯一默认发件账号，统一邮箱写信优先使用该账号，默认账号移除后由剩余账号自动接管。移除账号要求输入完整邮箱地址确认，至少保留一个账号，并在事务中级联清理本地邮件、目录、身份、信任、IMAP/OAuth 和发件箱关联数据，完成后自动切换到剩余账号。账号范围菜单使用名称 + 服务商/邮箱双行信息，同名账号也可区分。支持统一邮箱/单账号范围切换、写信发件账号选择、发件身份/别名管理、Reply-To 和身份级签名、统一邮箱下按待同步优先级聚合扫描多账号、IMAP/SMTP 主机、认证方式（应用专用密码/授权码或 OAuth2 Token）、OAuth2 PKCE 授权页启动、授权会话恢复记录、回调授权码记录、授权码 token 交换入 Keychain、refresh token 自动刷新和 XOAUTH2 登录、同步策略、账号级远程图片策略、按发件人/域名持久化的远程图片信任列表、签名。
- Gmail、Outlook、QQ 邮箱、网易 163 常见 IMAP/SMTP 服务商预设和兼容性矩阵，并按服务商预填推荐认证方式；网易 163 存在脱敏真实账号 TLS 与授权码认证历史通过样本，也记录了后续授权码被服务端拒绝的样本。新授权码必须仅在应用本地录入并重新验证，不能根据历史样本推断当前凭据仍然有效。
- 设置页可导出脱敏诊断 JSON，用于真实账号兼容性排查，不包含密码、token、授权 URL 或完整邮箱地址；也支持本地 JSON 备份/恢复，覆盖账号配置、文件夹、邮件、标签、附件元数据、规则、发件箱和同步记录，系统 Keychain 凭据不会写入备份。
- 设置页可按服务商记录真实账号验证结果：IMAP、SMTP、OAuth2/XOAUTH2、诊断导出状态和备注会保存在本地。
- 系统 Keychain 凭据保存/检查/删除，敏感信息不写入 SQLite。
- `mail-parser` MIME 解析、`ammonia` HTML sanitizer、受控 HTML 安全预览、正式阅读面安全渲染、默认阻止远程图片并支持信任发件人/域名后重渲染、阻止发件人并自动移入垃圾邮件、钓鱼链接域名不一致/IP/登录验证路径提示、附件名/数量识别。
- IMAP/SMTP 服务器网络测试、读取 Keychain 后的真实登录验证（SMTP 仅执行认证与 NOOP，不发送邮件）、IMAP 登录文件夹发现、OAuth2/XOAUTH2 登录、远端文件夹映射/UID 游标、邮件头增量同步、按同步策略触发的轻量后台同步、远端正文按需拉取。每个账号会顺序扫描已映射的收件箱、已发送、草稿、归档、废纸篓和垃圾邮件六类核心目录，每个目录单次最多拉取 25 条邮件头；单目录失败不会中断其他目录。自定义远端目录可在设置页直接选择现有本地自定义文件夹，或一键创建同名文件夹后加入增量同步；移动邮件到系统或已映射自定义目录时执行真实 `UID MOVE`，优先从 `COPYUID` 重绑定目标 UID，必要时按 Message-ID 定位，无法立即定位时由下次同步原地重绑定而不重复导入；恢复到收件箱、永久删除和清空废纸篓均会回写远端，清空操作按账号与远端目录批量执行 `\Deleted + UID EXPUNGE`；解除映射后会安全跳过同步且不再作为远端移动目标，避免误归入收件箱。
- 新邮件系统通知：后台/手动同步导入新邮件后申请系统通知权限并发送摘要，支持本地免打扰时段、VIP 发件人、仅 VIP 提醒、账号静音、重点账号提醒和系统未读角标，侧边栏展示最近同步、通知和角标状态。
- SQLite 持久化后台任务队列：手动/定时邮件头同步、发件箱发送演练和真实 SMTP 发送进入统一串行队列，任务状态可跨刷新/重启恢复，侧边栏展示最近任务状态；同类 queued/running 任务统一去重，延迟发送、稍后发送和失败重试只使用最早到期项的单个前端定时器触发。
- 正文拉取后会缓存附件元数据，附件文件可按需下载；真实 IMAP 下载优先用 BODYSTRUCTURE 定位 MIME part，再通过 BODY.PEEK part/chunk 分段写入本地 `.download` 临时文件，每个 BODYSTRUCTURE、整封回退和分段请求最多自动尝试 3 次，并在前两次失败后按 150 ms、300 ms 指数退避。编码后临时文件在 100 MB 安全范围内跨点击或应用重启保留，下次以文件长度作为 IMAP partial offset 续传；下载完成后根据 BODYSTRUCTURE 对 Base64、Quoted-Printable 或直通编码流式解码，解码后的最终文件仍严格限制为 25 MB。残缺或解码失败文件不会标记为已下载，阅读面显示已保留进度、明确原因与“重试”按钮；成功后原子重命名、记录本地路径、交给系统默认应用打开，并支持系统保存对话框另存为。
- 写信附件通过系统文件选择器或拖拽投放区记录文件元数据/可用本地路径，不把附件内容读入前端内存；附件会随草稿、发件箱和已发送消息保存元数据，富文本正文会清洗后保存为安全 HTML，并在发件箱真实 SMTP 发送时构建 `multipart/alternative` 或含附件的 `multipart/mixed` MIME。
- 发件箱队列、稍后发送、撤回到草稿箱、真实 SMTP 发送入口、OAuth2/XOAUTH2 发信认证、失败重试状态和 `next_attempt_at` 下次尝试时间。
- 联系人中心、联系人新建/编辑/删除、侧栏右键直达联系人编辑、别名/VIP 管理、手动合并、重复联系人合并建议、文件夹右键全部已读/清空废纸篓、自动规则执行、可视化规则构建器、规则新增/编辑/启停/删除、线程摘要和线程阅读视图等常见邮箱组织能力。
- Rust 数据层、协议、SMTP 和凭据边界单元测试，前端工具/服务商矩阵测试、Chrome headless UI 回归、桌面通知/角标权限配置、构建与 Clippy 验证。

## 开发命令

```bash
npm install
npm run build
npm run bench
npm run bench:release
npm run bench:app
npm run bench:sync
npm run probe:provider -- --list
npm run probe:provider -- --account-id 2
npm test
npm run test:ui
cd src-tauri && cargo test
cd src-tauri && cargo clippy --all-targets -- -D warnings
npm run tauri:dev
```

`probe:provider` 以只读方式打开现有数据库，并按账号 ID 从系统 Keychain 读取凭据；它只执行 IMAP 登录、SMTP 认证 + `NOOP`、文件夹发现和收件箱邮件头抓取，不发送邮件、不下载正文、不修改远端邮件状态。账号列表和 JSON 报告只输出脱敏邮箱、域名、服务商、阶段状态、耗时和数量；先在应用设置中保存账号与凭据，再用 `--list` 获取账号 ID。

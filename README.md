# Better Email

Better Email 是一个桌面端邮箱客户端实验。

它不把邮箱改成聊天，也不把邮件流做成信息流。主界面仍然是三栏：文件夹、邮件列表、阅读面板。常用动作留在手边，低频设置在需要时出现。

项目采用 Tauri、Rust 和 SQLite。邮件、联系人、规则、发件箱、同步记录和附件元数据保存在本地。账号密码和 OAuth token 不进入 SQLite，而是交给系统 Keychain。

Better Email 的目标不是一次做完所有邮箱功能。它先把本地工作台做稳，再逐步接入真实 IMAP、SMTP 和 OAuth2 流程。

设计文档见 [docs/DESIGN.md](docs/DESIGN.md)，验证记录见 [docs/VALIDATION.md](docs/VALIDATION.md)。

## 当前状态

本地工作台已经可用。

应用支持多账号、文件夹、邮件列表、会话阅读、搜索、标星、已读、归档、删除、移动、标签和稍后处理。常用动作放在阅读区和列表区。整理、安全和回写相关动作按组收进菜单。

撰写路径保持简单。

默认只显示收件人、主题和正文。发件身份、抄送、密送、富文本、签名、模板、附件和稍后发送放在工具区。草稿会本地保存。发送前可以设置撤销延迟，到期后再进入 SMTP 发件队列。

附件按需处理。

阅读邮件时只先保存附件元数据。用户点击后再下载文件。真实 IMAP 下载优先使用 BODYSTRUCTURE 定位 MIME part，并通过 BODY.PEEK 分段写入临时文件。失败时保留进度，用户可以重试。完成后再解码并写入最终文件。

安全默认值偏保守。

HTML 邮件会经过 sanitizer。远程图片默认阻止。用户可以按发件人或域名信任图片；信任后仍只放行 HTTPS 图片，且不会顺带恢复普通远程链接。凭据写入系统 Keychain。诊断导出会脱敏，不包含密码、token、授权 URL 或完整邮箱地址。

同步被拆成可验证的步骤。

认证页提供只读验收：服务器连接、系统凭据、IMAP 登录、SMTP 认证、文件夹发现和首轮邮件头同步。它不发送邮件，也不修改远端状态。SMTP 单独失败时，仍会继续验证 IMAP 收信链路。

发送与回写验收放在同步页。

应用只生成带验证编号的草稿。用户检查后手动发送。之后可以按编号追踪 SMTP、已发送留档、自发自收、附件读取和远端回写状态。

界面按桌面工作台设计。

首屏只加载三栏、列表和阅读面板。写信窗口、设置、命令面板和快捷键帮助按需加载。设置页按账号、认证、发送、通知、隐私、身份、备份、同步、联系人、规则和安全预览拆分，一次只渲染当前页。

## 技术边界

- 桌面壳：Tauri
- 后端：Rust
- 本地存储：SQLite
- 凭据：系统 Keychain
- 邮件协议：IMAP / SMTP / OAuth2 / XOAUTH2
- MIME 解析：mail-parser
- HTML 清洗：ammonia
- 前端：React + CSS，低频模块按需加载

## 已知边界

Better Email 仍是实验项目。

真实服务商兼容性还在补样本。附件下载、远端已读、星标、移动、恢复、永久删除、清空废纸篓、草稿同步和已发送留档已经有入口，但需要继续用 Gmail、Outlook、QQ、网易 163 和自建邮箱验证。

OAuth2 流程已有 PKCE、token 交换、refresh token 刷新和 XOAUTH2 登录路径。不同服务商的实际表现仍需要持续记录。

CardDAV、服务商联系人同步和更完整的恶意附件扫描不在当前阶段。

## 官网 / 作品集介绍

Better Email 是一个本地优先的桌面邮箱客户端实验。

它保留邮箱原来的工作方式：文件夹、邮件列表、阅读面板。没有聊天气泡，也没有信息流。常用动作在手边，低频设置在需要时出现。

数据先留在本机。

账号、邮件、联系人、规则、附件元数据和同步记录写入 SQLite。敏感凭据交给系统 Keychain。远程图片默认不加载。HTML 邮件先清洗，再渲染。

写信不抢注意力。

默认只保留收件人、主题和正文。身份、抄送、密送、签名、模板、附件和稍后发送放进工具区。用户可以专注把邮件写完，再处理格式和发送策略。

同步按风险分层。

只读诊断只检查连接、凭据、IMAP、SMTP、文件夹和邮件头。它不发送邮件，也不改远端状态。

写入验收需要用户确认。

应用生成带编号的草稿。用户检查后手动发送。之后再追踪 SMTP、已发送留档、自发自收、附件读取和远端回写。

附件按需下载。

邮件列表和阅读面不会预先拉取大文件。附件点击后才下载。IMAP 下载支持分段写入、失败重试和断点续传。未完成的文件不会被标记为可用。

这是一个仍在校准的项目。

本地工作台已经成形。真实服务商兼容性还在继续验证，尤其是附件、草稿、已发送留档、远端回写、OAuth2 和复杂 HTML 邮件。

## 开发命令

```bash
npm install
npm run build
npm run bench
npm run bench:release
npm run bench:app
npm run bench:sync
npm run probe:provider -- --list
npm run probe:provider -- --account-id 2
npm test
npm run test:ui
cd src-tauri && cargo test
cd src-tauri && cargo clippy --all-targets -- -D warnings
npm run tauri:dev
```

`probe:provider` 以只读方式打开现有数据库，并按账号 ID 从系统 Keychain 读取凭据。它只执行 IMAP 登录、SMTP 认证 + `NOOP`、文件夹发现和收件箱邮件头抓取，不发送邮件、不下载正文、不修改远端邮件状态。

账号列表和 JSON 报告只输出脱敏邮箱、域名、服务商、阶段状态、耗时和数量。先在应用设置中保存账号与凭据，再用 `--list` 获取账号 ID。
